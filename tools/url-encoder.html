URL encoder & decoder

Switch between component encoding (encodeURIComponent) and full URL encoding (encodeURI) to match your use case.

Full URL mode (keep / : ? # & structure)

Encode: component — spaces become %20, unicode is percent-encoded.

How to use

  1. Paste a URL or a string that should be encoded/decoded.
  2. Pick the mode that matches your use (component vs full URL).
  3. Copy the transformed value into your code or browser bar.

Use cases

Advanced Usage & Examples

Scenario 1: Safe redirect parameter in auth flow

Input value:
/checkout?step=2&plan=pro

As query param:
redirect=%2Fcheckout%3Fstep%3D2%26plan%3Dpro

Final URL:
https://app.example.com/login?redirect=%2Fcheckout%3Fstep%3D2%26plan%3Dpro

Scenario 2: Search query with special characters

Raw:
email=test+alerts@example.com & status=active

Encoded component:
email%3Dtest%2Balerts%40example.com%20%26%20status%3Dactive

Use in URL:
https://api.example.com/search?q=email%3Dtest%2Balerts%40example.com%20%26%20status%3Dactive

Did you know? 🧠

Double encoding usually happens when one layer already encoded the value and another layer repeats it. If you see %2520, the value was probably encoded twice.

Limitations

Encoding fixes URL safety, not logic correctness. A valid encoded URL can still point to the wrong route or include invalid business values.

How this differs from Base64 Encoder: URL encoding preserves web-safe transport in URLs, while Base64 converts binary/text payloads into a compact ASCII representation for storage or transfer.

Which mode should I use?

Component — for a single query value, path segment, or any string that must become “fully” escaped. Full URL — when you need to encode a mostly valid URL but fix unsafe characters in the path or query without breaking :// and delimiters.

Decoding must use the matching mode, or you may get a URI error.

Common real-world uses

URL encoding is useful when building query strings, passing redirect targets between services, cleaning tracking parameters, or preparing links for documentation and support tickets. It is also handy when debugging why an API request breaks after spaces, ampersands, equals signs, or Unicode characters enter a parameter value.

Avoid the most common mistake

The usual mistake is encoding an entire URL with encodeURIComponent, which can over-escape separators and make the final link unusable. Use component mode only for the part you are inserting, such as a search term, callback URL value, or path segment.

Quick encoding checklist

  • Encode only the fragment or value that actually needs escaping.
  • Use full URL mode when preserving separators matters.
  • Decode with the same logic that was used to encode.
  • Test the final URL in the actual app or request flow, not just in isolation.
