Sort text lines

Order every line A → Z or Z → A using locale-aware comparison.

Ignore case when sorting (recommended for mixed lists)

How to use

  1. Paste lines (one row per item).
  2. Toggle ignore-case if your list mixes capitalization.
  3. Sort A→Z or Z→A, then copy the result.

Use cases

  • Glossaries & tags

    Alphabetize before publishing.

    Spot duplicates after sorting.

Open Text & content tools

Locale & stability

Sorting uses localeCompare with numeric collation off. Line breaks from Windows (\r\n) and Unix (\n) are handled; result always uses \n between lines.

When this tool is useful

Alphabetical sorting is handy for keyword lists, tag sets, exports, email lists, inventory names, glossary terms, or any pasted content where you need to spot gaps, duplicates, and ordering mistakes quickly.

Better workflow for messy lists

If your source text is noisy, remove duplicates first, normalize casing, and then sort. That sequence usually produces cleaner output than sorting a mixed list with repeated or inconsistent entries.

Quick checklist

  • Use case-insensitive sorting for mixed editorial lists.
  • Deduplicate first if the order looks unexpectedly repetitive.
  • Normalize spacing before final copy-paste into docs or CMS fields.
