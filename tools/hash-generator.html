SHA-256 & SHA-1 hash generator

Fingerprint any UTF-8 text as lowercase hexadecimal. Uses the browser’s SubtleCrypto API — not for file checksums of large binaries (paste text only).

How to use

  1. Paste the text you want to digest.
  2. Choose SHA-256 or SHA-1 (hex output).
  3. Copy the hash for checksums or quick comparisons.

Use cases

  • Integrity checks

    Compare hashes before and after a transfer.

  • Cache keys

    Derive deterministic fingerprints from strings.

  • Pair with other tools

    Hash JSON after formatting to spot silent edits.

Input:

Stable deterministic ID for non-security test fixtures.

Did you know? 🧠

Any tiny input change (even one space) creates a completely different hash output due to the avalanche effect.

Limitations

This tool computes plain hashes, not keyed authentication (HMAC) and not digital signatures. Use cryptographic signing when authenticity must be proven.

How this differs from Password Generator: Hash Generator transforms existing input deterministically; Password Generator creates new random secrets.

Security notes

SHA-1 is deprecated for certificates and signatures, but still used for quick non-cryptographic checksums. Prefer SHA-256 for new workflows.

This page requires a secure context (https:// or localhost) for crypto.subtle in most browsers.

Your input is never uploaded — hashing runs entirely on your device.

What a hash is good for

A cryptographic hash creates a deterministic fingerprint for a piece of text. That makes it useful for integrity checks, deduplicating values, comparing secrets without storing the original plain text, or confirming whether two strings are exactly identical.

What it is not good for

Hashing is not the same as encryption. You cannot recover the original text from a proper hash, and plain unsalted hashes are not suitable for storing user passwords in production. For password storage, use a slow password hashing algorithm like Argon2, bcrypt, or scrypt on the server.

Quick verification checklist

  • Use SHA-256 for new workflows unless a legacy system explicitly requires SHA-1.
  • Compare hashes only when the input encoding is consistent.
  • Remember that even a one-character change produces a different digest.
  • Do not use this page for large file checksums, since it hashes pasted text rather than binary file contents.
