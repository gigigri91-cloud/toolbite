Case converter

Switch between UPPERCASE, lowercase, Title Case, Sentence case, and camelCase — ideal for copy, code, and SEO titles.

How to use

  1. Paste or type your text.
  2. Tap UPPERCASE, Title Case, camelCase, or another mode.
  3. Copy the converted text or clear and start over.

Use cases

  • Headlines & UI labels

    Normalize titles before publishing.

  • Code & JSON keys

    Switch to camelCase for variables and APIs.

  • Imported copy

    Fix inconsistent capitalization in one pass.

Did you know? 🧠

Automated case conversion rarely handles acronyms perfectly (API, JWT, SEO). A quick human review keeps brand and technical terminology correct.

Limitations

Case rules vary by style guide and language. This tool applies generic rules, so edge cases like proper nouns may need manual refinement.

How this differs from Text to Slug: Case Converter preserves readable text variants; Text to Slug transforms phrases into URL-safe, hyphenated slugs.

Open Text & content tools

When to use each case style

Title Case is common for blog headlines. Sentence case reads naturally in paragraphs. camelCase matches JavaScript variables and JSON keys. All transformations run locally so drafts stay private.

Common editing scenarios

Case conversion is useful when cleaning imported headlines, standardizing UI labels, fixing inconsistent capitalization in lists, preparing JSON keys, or turning rough notes into publishing-ready titles and headings.

Where mistakes usually happen

Title Case is often overused in body copy, while Sentence case is sometimes forgotten in UI strings or metadata drafts. camelCase is convenient for variables but rarely appropriate for reader-facing text. Picking the right mode saves cleanup later in the workflow.

Simple workflow tip

Convert the text first, then check slug, length, and final wording with related tools. That works better than generating a slug or word count from a heading that still has broken capitalization.
